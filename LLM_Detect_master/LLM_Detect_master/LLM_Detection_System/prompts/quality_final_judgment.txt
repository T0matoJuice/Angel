# 判断决策逻辑

```
对于每一条数据：
    1. 检查"服务项目或故障现象"
        - 包含"质量鉴定" → 质量工单（高置信度）
        - 包含"安装/调试/加装" → 非质量工单（高置信度）
        - 其他情况 → 进入下一步

    2. 分析"判定依据"
        - 指向产品缺陷 → 累计+30%质量权重
        - 指向环境/人为 → 累计-30%质量权重

    3. 参考"故障类别"
        - 根据训练数据中该类别的分布调整权重±15%

    4. 检查"旧件名称"vs"新件名称"
        - 不同且非耗材 → +10%质量权重

    5. 综合"维修问题点"和"二级问题点"
        - 分析问题本质 → 调整权重±5%

    6. 最终判断
        - 累计权重 > 0 → 质量工单
        - 累计权重 ≤ 0 → 非质量工单

    7. 生成判定依据
        - 质量工单：简要说明产品问题
        - 非质量工单：按5要素模板生成详细依据
```

# 特殊情况处理

- 服务项目="质量鉴定"，但判定依据="环境问题"

## 信息不足

  → 参考训练数据中信息不足案例的整体分布倾向

## 边界模糊
- 既有产品问题又有环境问题
  → 判断主要原因，参考训练数据中类似复杂案例
