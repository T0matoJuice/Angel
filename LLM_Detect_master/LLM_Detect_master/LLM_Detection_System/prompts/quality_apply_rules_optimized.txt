# 判断流程（6步判断法）

## 第1步：字段完整性检查
- 检查每条数据的字段完整性
- 识别缺失关键信息的记录
- 标记需要综合判断的边界案例

## 第2步：一级字段判断（服务项目或故障现象）权重40%
**强判定规则**（参考训练数据的规律）：
- ✅ "质量鉴定"、"产品质量鉴定" → 质量工单（99%置信度）
- ❌ "安装"、"调试"、"加装"、"改装"、"移机" → 非质量工单（95%置信度）
- ⚠️ "维护"、"保养"、"清洗" → 倾向非质量工单（80%置信度）

**操作要点**：
- 优先识别该字段的明确指示
- 如果该字段有明确标识，可直接做出高置信度判断
- 如果该字段为空或模糊，进入第3步综合判断

## 第3步：二级字段判断（判定依据）权重30%
**分析要点**：
- 质量倾向：判定依据指向产品缺陷（"零部件损坏"、"失效"、"性能不达标"）
- 非质量倾向：判定依据指向环境/人为（"水压"、"水质"、"电压"、"操作不当"、"安装问题"）

**操作要点**：
- 深度理解判定依据的语义，而非表面关键词
- 识别因果关系：问题根源是产品还是外部因素？
- 参考训练数据中类似表述的标注结果

## 第4步：三级字段判断（故障类别）权重15%
**参考训练数据的分布规律**：
- 哪些故障类别在训练数据中倾向质量工单？
- 哪些故障类别在训练数据中倾向非质量工单？

**操作要点**：
- 回忆训练数据中该故障类别的标注倾向
- 结合其他字段综合判断

## 第5步：四级字段判断（部件更换）权重10%
**特殊规则**：
- 如果"旧件名称"≠"新件名称"且都非空 → 质量怀疑权重+20%
- 但需注意：滤芯、耗材的正常更换不算质量问题

**操作要点**：
- 识别是否存在部件更换行为
- 判断更换原因：质量问题 vs 正常损耗
- 参考训练数据中类似更换案例的标注

## 第6步：辅助字段综合判断（权重5%）
**来电内容、现场诊断、处理方案**：
- 提取关键词和短语
- 分析整体语义倾向
- 寻找训练数据中的相似案例

# 判断决策树

```
开始判断
    ↓
[第1步] 服务项目或故障现象 是否明确？
    ├─ 是 → 包含"质量鉴定"？
    │        ├─ 是 → 【质量工单】(99%置信度)
    │        └─ 否 → 包含"安装/调试/加装"？
    │                 ├─ 是 → 【非质量工单】(95%置信度)
    │                 └─ 否 → 进入第2步
    └─ 否 → 进入第2步
        ↓
[第2步] 判定依据 是否指向产品缺陷？
    ├─ 是 → 累计权重+30% → 进入第3步
    └─ 否 → 指向环境/人为？
            ├─ 是 → 累计权重-30% → 进入第3步
            └─ 否 → 进入第3步
        ↓
[第3步] 故障类别 在训练数据中的倾向？
    ├─ 质量倾向 → 累计权重+15% → 进入第4步
    └─ 非质量倾向 → 累计权重-15% → 进入第4步
        ↓
[第4步] 部件更换情况 是否存在？
    ├─ 旧件≠新件 → 累计权重+10% → 进入第5步
    └─ 无更换 → 进入第5步
        ↓
[第5步] 辅助字段 语义倾向？
    ├─ 质量倾向 → 累计权重+5%
    └─ 非质量倾向 → 累计权重-5%
        ↓
[第6步] 最终判断
    ├─ 累计权重 > 0 → 【质量工单】
    └─ 累计权重 ≤ 0 → 【非质量工单】
```

# 特殊情况处理

## 情况1：信息严重缺失
- 如果关键字段（服务项目、判定依据、故障类别）全部为空
- 参考训练数据中信息不足案例的标注分布
- 基于仅有信息进行保守判断

## 情况2：字段信息冲突
例如：服务项目="质量鉴定"，但判定依据="水压问题"
- 以"服务项目或故障现象"为最高优先级（权重40%）
- 但需在判定依据中说明矛盾点

## 情况3：边界模糊案例
例如：因安装问题导致的产品损坏
- 判断主要原因：安装问题 → 非质量工单
- 判断主要原因：产品缺陷在先 → 质量工单
- 参考训练数据中类似复杂案例的标注逻辑

## 情况4：多因素混合
- 按权重体系累计计算
- 参考训练数据中的整体标注倾向
- 选择累计权重更高的一方

4. **置信度评分**：给出0-100%的置信度

   - 环境因素类：75-85%
   - 使用不当类：70-80%
   - 信息不足类：50-65%

5. **证明材料或说明文档**（可选但建议）：
   - 例如："建议核实：客户安装记录、现场诊断报告、环境检测数据"
